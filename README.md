# AI + PQC Security Monitoring Streamlit Demo V2

AI anomaly detection dashboard with a simulated post-quantum cryptography response policy.

## Quick Start

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Current Workflow

1. Upload a training CSV.
2. Adjust `contamination` and random seed settings.
3. Click `Train Model`.
4. Prepare test data by either:
   - simulating test rows from the uploaded training CSV,
   - uploading a separate test CSV,
   - simulating test rows from the uploaded test CSV.
5. Choose a risk-response threshold mode:
   - `auto`: use the Isolation Forest internal threshold learned from `contamination`.
   - `manual`: use adjustable risk thresholds and medium-risk streak count.
6. Click `Run Test`.

Training data is not generated by the app. The model is trained only from the uploaded training CSV.

## Risk Response Strategy

The app maps Isolation Forest scores into a normalized risk value in `[0, 1]`. Higher risk means more anomalous. The risk scale is calibrated from the uploaded training CSV, then reused for test data so threshold values are more stable across test batches.

Threshold modes:

- `auto`: no manual risk grading. The app uses Isolation Forest's own `predict()` result, whose anomaly threshold comes from the trained model's `contamination` setting.
- `manual`: use the adjustable risk-response policy below.

Manual configurable parameters:

- `Low/Medium risk threshold`
- `Medium/High risk threshold`
- `Medium-risk streak trigger count`

Manual default values:

- `Low/Medium risk threshold = 0.015`
- `Medium/High risk threshold = 0.20`
- `Medium-risk streak trigger count = 5`

These manual defaults are starting values for the manual policy. After changing the feature set or traffic mix, retune them against a held-out labeled test set. The `auto` mode is the safer default because it uses the Isolation Forest threshold learned from the current `contamination` setting.

Manual policy:

- Low risk: pass directly.
- Medium risk: monitor; trigger simulated key renegotiation only after `n` consecutive medium-risk samples.
- High risk: immediately trigger simulated key renegotiation.

False positive rate and false negative rate are calculated from the final response decision:

- Response triggered = anomaly prediction.
- No response triggered = normal prediction.

If the test data has no `label` column, false positive rate and false negative rate are shown as `N/A`.

## PQC Scope

The PQC module is a policy simulation.

It simulates switching between:

- `Kyber512` for normal/pass behavior.
- `Kyber768` for high-risk or medium-streak response behavior.

It does not perform a real liboqs/Kyber key exchange. The renegotiation count is a simulated response metric.

## CSV Format And Features

The app keeps a low-dimensional but more representative feature set for CICIDS2017-style traffic. It does not use all CICIDS columns.

Base internal feature columns:

- `duration`
- `packet_count`
- `byte_size`
- `src_bytes`
- `dst_bytes`
- `flow_rate`

When CICIDS2017 columns are present, the loader also uses these representative features:

- `flow_packets_s`
- `fwd_packets_s`
- `bwd_packets_s`
- `flow_iat_mean`
- `flow_iat_std`
- `fwd_iat_mean`
- `bwd_iat_mean`
- `packet_length_mean`
- `packet_length_std`
- `syn_flag_count`
- `ack_flag_count`
- `rst_flag_count`
- `psh_flag_count`
- `down_up_ratio`
- `init_win_bytes_forward`
- `init_win_bytes_backward`
- `active_mean`
- `idle_mean`

Optional:

- `label`

Label convention:

- `-1` = anomaly
- `1` = normal

The loader recognizes common CICIDS2017-style column names and trims extra spaces in headers. Generic CSVs with only the six base features are still supported.

## Training Preprocessing

The model applies preprocessing inside the trained detector:

1. Convert non-numeric feature values to missing values.
2. Replace `inf` and `-inf` with missing values.
3. Fill missing values with training-set medians.
4. Clip extreme values using training-set quantile bounds.
5. Apply `log1p` to long-tailed traffic features such as bytes, packet counts, rates, IAT, active/idle timing, and packet length statistics.
6. Scale features with `RobustScaler`.

The preprocessing parameters are fitted only on the training data and then reused for test data. Test data is not used to fit preprocessing parameters.

If the training CSV contains a `label` column, only normal/BENIGN rows are used for fitting the one-class Isolation Forest model. This matches the CICIDS2017 workflow where Monday traffic is normal training data and the other files are mixed normal/attack test data.

## Test Data Simulation

Simulated test data is created from an uploaded CSV by:

1. Resampling rows from that CSV with replacement.
2. Adding light noise only to base flow features: `duration`, `packet_count`, `src_bytes`, and `dst_bytes`.
3. Recomputing dependent features:
   - `byte_size = src_bytes + dst_bytes`
   - `flow_rate = byte_size / duration`
4. Preserving labels from the sampled rows if labels are present.

This is useful for robustness testing against small perturbations. It is not a substitute for evaluation on a real held-out test CSV.

## Features

- CSV-only model training.
- Separate training and testing data flow.
- Test-data simulation from uploaded training or test CSV.
- `contamination` parameter control, including `auto`.
- Fixed or dynamic random seed.
- Risk-level response thresholds.
- Medium-risk consecutive trigger policy.
- Stop button for live testing.
- False positive rate and false negative rate.
- English/Chinese UI switch.
- Large-test UI safeguards: full metrics are computed on all rows, while tables, charts, and page logs are display-limited for responsiveness.
- Plotly charts and event log.

## Upgrades From The Initial Version

The current version extends the original demo in these areas:

- Training data is now provided by uploaded CSV files only. The app no longer creates its own synthetic training set.
- A dedicated `Train Model` button was added under the training CSV section, so users can retrain after changing `contamination` or random seed settings without re-uploading the CSV.
- Test data can come from three paths: simulated from the training CSV, uploaded as a separate test CSV, or simulated from the uploaded test CSV.
- Uploaded test CSV data can be restored after simulated test data is generated.
- Live testing now includes a stop button.
- False positive rate and false negative rate are calculated when labeled test data is available.
- The sidebar now supports fixed or dynamic random seed modes.
- The `contamination` parameter can be set to `auto` or a custom numeric value before training.
- Risk response supports two threshold modes:
  - `auto`, which uses the trained Isolation Forest threshold from `contamination`.
  - `manual`, which uses adjustable low/high risk thresholds and a medium-risk streak trigger.
- The manual response policy was changed to a three-level strategy: low risk passes, medium risk triggers only after consecutive occurrences, and high risk triggers immediate simulated key renegotiation.
- The UI supports English/Chinese switching.
- The Detection Results metric styling was adjusted for better contrast.
- CICIDS2017 support was expanded from six basic features to a compact 24-feature representative set.
- Training now includes a model-level preprocessing pipeline: missing/invalid value handling, extreme-value clipping, `log1p` transformation, and `RobustScaler` scaling.
- Preprocessing parameters are fitted only on training data and reused for test data to avoid data leakage.
- If training data includes labels, only normal/BENIGN rows are used to fit the one-class Isolation Forest model.
- Large-result rendering was optimized so full-test metrics still use all samples, while the UI only displays a limited number of rows, chart points, and log entries.
